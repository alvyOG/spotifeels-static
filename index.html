Big Questions

  1. Q: Can we develop a search engine that allows a user to generate a playlist from a given search query?
  2. A: More or less, using Spotify API and Genius API, in combination with some IR features!
  3. Q: Can we generate a song lyric index for recent, popular songs?
  4. A: Yes! By pulling songs from the Billboard Top 200 (Albums) via Spotify API, and their lyrics via Genius API, we can compile an index while also making use of tokenization and stemming.
  5. Q: Can we search through the index using terms in the query?
  6. A: Yes! We can perform a direct text match between the query terms and lyrics using TF-IDF to get the most relevant results.
  7. Q: Can we perform sentiment analysis on the lyrics and query to better match the playlists ideal sentiment?
  8. A: Yes, we have identified several libraries and lexicons to allow us to do this, in addition to Spotify's song (valence) features. This is a key future implementation!

Key Takeaways

Negatives

  1. Multiple Languages in the Index can Cause Stemming, Multi-Index Issues
  2. Conflicting Sentiment in a Query Allows for Both Sentiments in Results
  3. Direct Text Match Abandons Context

Positives

  1. Indexing Song Lyrics Allows More Specificity in Search Queries
  2. Combining Various Sentiment Factors Allows for Larger Sentiment Space
